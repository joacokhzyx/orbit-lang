# Orbit Self-Hosting Specification
This document defines how Orbit compiles itself to achieve complete independence.

## Stages of Self-Hosting

The compiler bootstrap process is organized into distinct, verifiable stages:

```
                  [Stage 0 Compiler (Zig + Steel)]
                                │
                                ▼
         Compiles Compiler Source (written in Orbit)
                                │
                                ▼
                  [Stage 1 Compiler (Native/Steel)]
                                │
                                ▼
                Compiles Compiler Source again
                                │
                                ▼
                  [Stage 2 Compiler (Native)]
                                │
                                ▼
                Compiles Compiler Source again
                                │
                                ▼
                  [Stage 3 Compiler (Native)]
```

*   **Stage 0**: The original compiler binary built with the Zig toolchain using the Steel (C backend) path.
*   **Stage 1**: The first compiler executable generated by compiling the Orbit-written compiler source code using the Stage 0 compiler.
*   **Stage 2**: The compiler executable generated by compiling the compiler source using the Stage 1 compiler.
*   **Stage 3**: The compiler executable generated by compiling the compiler source using the Stage 2 compiler. At this stage, Stage 2 and Stage 3 must reach a **fixed point** where they are functionally and semantically identical.
